# User Manual – veda_ped_analyzer

## 1. Overview

This program parses PED output generated by VEDA and produces CSV tables.
Optionally, vibrational data from ORCA or Gaussian outputs are used
to align vibrational modes.

## 2. Input files

### 2.1 VEDA `.ved`
Contains PED (and optionally TED) matrices and vibrational frequencies.

### 2.2 VEDA `.dd2`
Defines internal coordinates used to label PED columns.

### 2.3 Quantum-chemistry output

Used to extract:
- Vibrational frequencies
- IR intensities
- Symmetry labels (if present)
- Atom information for labeling

Supported formats:
- ORCA `.out`
- Gaussian `.log` / `.out`

## 3. Output files

### 3.1 `*_PED_table.csv`

One row corresponds to one vibrational mode.

Important columns:
- `mode_veda`, `freq_veda`
- `mode_qc`, `freq_qc`
- `delta_freq`, `abs_delta_freq`
- `irrep`
- `IR_intensity`
- `PED1_*` … `PED6_*`

### 3.2 `*_coordinates_lookup.csv`

One row per internal coordinate.
The `top_mode_terms` column summarizes dominant modes, e.g.:

`freq=1613:22.0; freq=1639:18.0`

## 4. Interpretation notes

- PED values are normalized to 100%.
- Mode mixing is common, especially for symmetric molecules.
- Always verify assignments when frequency differences are large.

## 5. Troubleshooting

- Ensure VEDA and QC outputs correspond to the same geometry.
- Close Excel before rerunning if CSV saving fails.
