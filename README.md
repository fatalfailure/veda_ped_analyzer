# veda_ped_analyzer

**Current version:** v1.1.9  
**Repository:** https://github.com/fatalfailure/veda_ped_analyzer

`veda_ped_analyzer` is a GUI-based Python tool for analyzing **Potential Energy Distribution (PED)** data generated by **VEDA** (`.ved`, `.dd2`). It can correlate VEDA vibrational modes with quantum-chemistry vibrational outputs from **ORCA** or **Gaussian** and export publication-ready CSV tables.

The program is intended as a **general PED analysis tool**. Version 1.1.x adds a target-coordinate workflow: users can select any internal coordinates of interest and trace their PED contributions across all normal modes. Metal-ligand stretch auto-detection is available only as an optional helper for coordination-complex studies; it is not the default analysis mode.

## Main features

- GUI workflow for VEDA `.ved` / `.dd2` PED analysis
- ORCA and Gaussian vibrational-output parsing
- Frequency-based VEDA-QC mode alignment
- Standard mode-centered top-N PED table export
- Coordinate Browser for DD2 internal coordinates
- Target-coordinate tracking for selected internal coordinates
- Optional metal-ligand stretch auto-detection helper
- Long-format PED table export for searching contributions below the usual top-N cutoff
- Target hits, target summary by mode, target summary by coordinate, and target matrix exports
- Optional `alternative_k` / `alternative_v` coordinate-set export, disabled by default
- Combined target outputs that can safely sum mixed `s` / `k` / `v` target references

## Requirements

- Python 3.9 or later recommended
- `pandas`
- Standard-library `tkinter`

Install the required package with:

```bash
pip install pandas
```

## Input files

Required:

- VEDA `.ved`
- VEDA `.dd2`
- Quantum-chemistry vibrational output:
  - ORCA: `.out`
  - Gaussian: `.log` or `.out`

Optional:

- `.fmu`, used for improved atom labeling when available

## Running the program

```bash
python veda_ped_analyzer.py
```


## Coordinate Browser filter help

The Coordinate Browser now explains the two less obvious filters directly in the GUI.

- **Atom filter** filters by atom index. `3` shows coordinates containing atom 3. `3,12` shows coordinates containing atom 3 **or** atom 12. `3-12` shows coordinates containing **both** atoms 3 and 12, which is useful for bond/pair searches. Full-width and en-dash variants such as `3－12` and `3–12` are also accepted.
- **Text filter** searches the displayed atom label, VEDA label, raw label, and source line. Examples include `C4-C11`, `CC`, `ZnO`, and `str`.

## Coordinate Browser defaults

The Coordinate Browser opens with these default filters:

- **Coordinate set:** `s - standard`
- **Group:** `STRE - Stretching`

This makes it easier to start from the standard stretching-coordinate definitions. The coordinate-set selector uses the VEDA/DD2 code:

| Display | Meaning |
|---|---|
| `s - standard` | Standard internal-coordinate interpretation |
| `k - alternative` | Alternative internal-coordinate interpretation |
| `v - alternative2` | Second alternative internal-coordinate interpretation |
| `(any)` | Show all available coordinate sets |

## Target set name

`target_set_name` in output CSV files is only a **user-defined label** for the selected coordinate group. It is not a calculation mode and does not imply that the analysis is limited to metal complexes.

The default value is:

```text
target_coordinates
```

For example, users may set it to `ring_CC_stretches`, `carbonyl_modes`, `Co_N_stretches`, or any other meaningful label. If the optional metal-ligand auto-detection button is used, the name may be set to `metal_ligand_stretch` as a descriptive label for that selected target set.


### Code-qualified target coordinates

Targets are stored as coordinate-set-qualified references such as `s:81`, `k:128`, or `v:81`. This avoids ambiguity when the same numeric `coord_id` has different meanings in the standard (`s`) and alternative (`k`/`v`) coordinate sets. Plain numeric target IDs are still accepted for backward compatibility, but new selections from the Coordinate Browser are written in the code-qualified format.

When **Include alternative coordinate sets (k/v)** is enabled, standard, alternative_k, and alternative_v outputs are generated separately. The standard results are therefore retained while the alternative coordinate sets can be inspected independently.

### Combined target outputs

Version 1.1.3 adds combined target outputs for cases where chemically related coordinates cannot be represented within a single VEDA coordinate set. For example, one ring bond may be available as `s:83`, while the corresponding opposite ring bond may only be available as `k:126`. In this case, add both references to the same target set and enable **Combined target outputs (allow mixed s/k/v targets)**.

The program then writes files such as:

- `sample_target_summary_by_mode_combined.csv`
- `sample_target_hits_combined.csv`
- `sample_target_summary_by_coord_combined.csv`
- `sample_target_matrix_combined.csv`

These files use the selected code-qualified references literally and keep `target_ref` in the output, so mixed standard/alternative coordinates remain traceable.

## Output files

For a file `sample.ved`, typical standard outputs include:

- `sample_PED_table_standard.csv`  
  Mode-centered top-N PED table.
- `sample_PED_terms_long_standard.csv`  
  Long-format table containing searchable PED contributors.
- `sample_target_hits_standard.csv`  
  Rows for target-coordinate contributions detected in each mode.
- `sample_target_summary_by_mode_standard.csv`  
  Mode-centered summary of the total PED contribution from the selected target-coordinate set.
- `sample_target_summary_by_coord_standard.csv`  
  Coordinate-centered summary showing where each selected coordinate appears.
- `sample_target_matrix_standard.csv`  
  Matrix form: rows are modes and columns are selected target coordinates.
- `sample_coordinates_lookup.csv`  
  DD2 coordinate lookup table.

### Optional alternative-coordinate outputs

`alternative_k` and `alternative_v` outputs are disabled by default. Enable **Include alternative coordinate sets (k/v)** in the Run Analysis tab only when you need to inspect alternative DD2/VEDA internal-coordinate interpretations.

## Common Windows issue: Permission denied

If `[Errno 13] Permission denied` occurs when saving CSV files, common causes are:

- The output CSV is open in Excel
- The output directory is write-protected
- A sync folder such as OneDrive is locking the file

Close Excel and select a writable output directory such as Desktop or Documents.


### Combined output matrix-source selection (v1.1.9)

For a mixed target set, each requested target reference (`s:*`, `k:*`, or `v:*`) is evaluated with the corresponding DD2 coordinate-set interpretation. If the `.ved` file does not contain an explicit alternative PED block, the available PED matrix, usually the standard matrix, is reused as the numeric source and interpreted with the requested DD2 coordinate labels. The columns `requested_target_refs`, `found_target_refs`, and `missing_target_refs` indicate whether all mixed targets were included.

If matrix-based matching still cannot find a requested target, the DD2-terms fallback may be used; such terms are marked with sources such as `dd2_terms_alternative_k`.
