# veda_ped_analyzer

`veda_ped_analyzer` is a GUI-based Python tool for analyzing **Potential Energy Distribution (PED)** data generated by **VEDA** (`.ved`, `.dd2`) and correlating the result with vibrational data from **ORCA** or **Gaussian** outputs.

The program was originally designed to export mode-centered PED tables. Version 1.1.0 adds a **target coordinate tracking** workflow, which is useful when a selected internal coordinate, such as a metal-ligand stretching coordinate in a coordination complex, is distributed across many normal modes in the fingerprint region.

## Version

Current release: **v1.1.0 - Target Coordinate Tracking**

## Repository

GitHub: https://github.com/fatalfailure/veda_ped_analyzer

## Main features

- GUI workflow for VEDA PED analysis
- ORCA and Gaussian vibrational-output parsing
- Frequency-based VEDA-to-QC mode mapping
- DD2 internal-coordinate browser
- Target coordinate tracking for selected internal coordinates
- Automatic detection of metal-ligand stretching coordinates
- Original top-N mode-centered PED table export
- Long-format PED table export for searching lower-rank PED terms
- Target hits export
- Target summary by mode export
- Target summary by coordinate export
- Target matrix export for spreadsheet heatmaps
- Optional `alternative_k` / `alternative_v` coordinate-set export
- JSON-based restoration of previous GUI settings

## Required inputs

Required:

- VEDA `.ved`
- VEDA `.dd2`
- Quantum-chemistry vibrational output:
  - ORCA: `.out`
  - Gaussian: `.log` or `.out`

Recommended:

- `.fmu` for improved atom-index to element-symbol mapping

If `.fmu` is not provided, the program attempts to infer atom labels from the QC output geometry block.

## Running the program

```bash
python veda_ped_analyzer.py
```

The GUI opens with the following tabs:

- `Files / Precheck`
- `Coordinate Browser`
- `Target Definition`
- `Run Analysis`
- `Results Preview`
- `User Guide`

## Basic workflow

1. Open `Files / Precheck` and select `.ved`, `.dd2`, optional `.fmu`, and QC output (`.out` / `.log`).
2. Press `Load / Precheck` and confirm that PED blocks, DD2 coordinates, QC frequencies, and mode mapping were detected.
3. Open `Coordinate Browser` and find the internal coordinates of interest.
4. Add selected internal coordinates to the target list, or use automatic metal-ligand stretch detection.
5. Open `Run Analysis`, set PED thresholds and the frequency range.
6. Press `RUN TARGET ANALYSIS`.
7. Inspect `Results Preview` and the exported CSV files.

## Output files

The output file prefix is derived from the input `.ved` file name.

### Standard output

- `*_PED_table_standard.csv`  
  Original-style mode-centered table with top-N PED contributors.

- `*_PED_terms_long_standard.csv`  
  Long-format table containing PED contributors by mode and internal coordinate. This is useful for finding target coordinates that are below the top-N cutoff.

- `*_target_hits_standard.csv`  
  List of modes where the selected target coordinates are detected.

- `*_target_summary_by_mode_standard.csv`  
  Mode-centered summary of the total PED contribution from the selected target-coordinate set. This is the main table for finding split metal-ligand stretching components.

- `*_target_summary_by_coord_standard.csv`  
  Coordinate-centered summary showing where each selected internal coordinate appears most strongly.

- `*_target_matrix_standard.csv`  
  Matrix format: rows are vibrational modes and columns are target coordinates.

- `*_coordinates_lookup.csv`  
  Lookup table of DD2 internal coordinates and simplified atom labels.

### Optional alternative-coordinate outputs

`alternative_k` and `alternative_v` outputs are disabled by default. Enable `Include alternative coordinate sets (k/v)` in the `Run Analysis` tab only when you need to inspect alternative DD2/VEDA internal-coordinate interpretations.

When enabled, files such as the following may also be generated:

- `*_PED_table_alternative_k.csv`
- `*_PED_terms_long_alternative_k.csv`
- `*_target_summary_by_mode_alternative_k.csv`
- `*_PED_table_alternative_v.csv`
- `*_PED_terms_long_alternative_v.csv`
- `*_target_summary_by_mode_alternative_v.csv`

## Important columns

### `*_PED_table_standard.csv`

- `mode_veda`, `freq_veda`
- `mode_qc`, `freq_qc`, `delta_freq`, `abs_delta_freq`
- `irrep`
- `IR_intensity`
- `PED1_*`, `PED2_*`, ... up to the selected top-N limit
- `total_target_PED`
- `top_target_terms`

### `*_target_summary_by_mode_standard.csv`

- `mode_qc`, `freq_qc`
- `IR_intensity`
- `total_target_PED`
- `max_target_PED`
- `n_target_coords_detected`
- `best_target_rank`
- `top_target_terms`

### `*_target_summary_by_coord_standard.csv`

- `coord_id`, `label`
- `max_PED`
- `mode_qc_at_max`, `mode_veda_at_max`
- `freq_qc_at_max`, `freq_veda_at_max`
- `sum_PED_in_range`
- `weighted_mean_freq`
- `n_modes_detected`
- `top_modes`

## Configuration and logs

The program saves previous GUI settings to:

```text
veda_ped_analyzer.json
```

If the program directory is not writable, the JSON file is saved in the user's home directory.

Log messages are saved to:

```text
veda_ped_analyzer.log
```

or to the user's home directory if needed.

These local `.json` and `.log` files should not normally be committed to Git.

## Common Windows issue: Permission denied

If you encounter `[Errno 13] Permission denied` when saving CSV files, common causes include:

- The output CSV is open in Excel
- The output directory is write-protected
- A cloud-sync folder restricts write access

Close Excel and choose a writable output folder such as Desktop or Documents.
